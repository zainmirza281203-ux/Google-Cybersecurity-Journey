# Course 2: Play It Safe: Manage Security Risks

## Course Objective
The primary focus of this course was to understand how organizations identify, assess, and manage security risks. I learned how to apply structured methodologies to maintain a strong security posture.

## Detailed Curriculum Highlights

### 1. Risk Management Frameworks
I explored how security teams use frameworks to categorize and handle threats systematically[cite: 1, 2]. This includes understanding the lifecycle of security incidents from identification to recovery[cite: 1, 2].

### 2. Incident Response (IR) Fundamentals
I mastered the six key phases of the Incident Response lifecycle:
* **Preparation:** Establishing procedures and training teams to minimize risk impact[cite: 1, 2].
* **Detection:** Identifying potential security incidents using monitoring tools[cite: 1, 2].
* **Analysis:** Evaluating the severity and origin of a threat[cite: 1, 2].
* **Containment:** Taking immediate actions to isolate affected systems and prevent further spread[cite: 1, 2].
* **Eradication:** Removing artifacts of the incident from the network[cite: 1, 2].
* **Recovery:** Restoring operations using clean backups[cite: 1, 2].

### 3. Security Tooling: SIEM and SOAR
I learned the practical distinction and synergy between security tools:
* **SIEM (Security Information and Event Management):** Used for monitoring network activity and generating alerts for potential threats[cite: 1, 2].
* **SOAR (Security Orchestration, Automation, and Response):** Used for automating repetitive security tasks to increase efficiency[cite: 1, 2].
* **Playbooks:** I learned that playbooks (or runbooks) are essential, living documents that provide step-by-step instructions for analysts to follow during a security breach[cite: 1, 2].

## Technical Competencies Gained
* **Operational Efficiency:** Utilizing automation to handle routine security alerts[cite: 1, 2].
* **Analytical Thinking:** Assessing log data to differentiate between false alarms and actual security incidents[cite: 1, 2].
* **Documentation & Compliance:** Ensuring incident response follows established organizational standards to maintain compliance[cite: 1, 2].
